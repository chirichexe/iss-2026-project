Firefly project: Task A

Introduction

Goal: Realizzazione in QAK di un sistema costituito da 3 lucciole le quali:
  • Si accendono e si spengono con frequenza costante
  • Dopo 10 secondi le lucciole lampeggiano in modo sincronizzato

Requirements

R1)  Realizzare un sistema software che simuli 3 lucciole su una griglia. 

R2)  Ogni lucciola lampeggia (cioè si accende e si spegne) con una frequenza costante, eventualmente diversa per ciascuna lucciola. 

R3)  Il sistema deve garantire la transizione automatica al lampeggio sincronizzato dopo 10 secondi dall'avvio del sistema.

Requirement analysis

  • All'interno del sistema le lucciole devono essere rappresentate da attori che lampeggiano con intervalli di tempo inizialmente variabili
  • Il comportamento di una lucciola è caratterizzato da una posizione (X,Y) sulla griglia, un timer interno e un ciclo di accensione e spegnimento

Problem analysis

Il ciclo di una lucciola può essere modellato come uno stato che alterna acceso/spento con un tempo variabile. 

	State flash {
		forward griddisplay -m cellstate: cellstate($X,$Y,1) // stato ON
		delayVar Timer // aspetta il suo tempo...
		forward griddisplay -m cellstate: cellstate($X,$Y,0) // stato OFF
	}
La ripetizione del ciclo è gestita da una transizione temporale: 

Transition t0
whenTimeVar Timer -> flash
Mediante la "sincronizzazione" introduciamo un evento globale e un riallineamento temporale basato su un tempo di riferimento comune 

Event syncronize : syncronize(TIME, FREQ)
Ogni attore, alla ricezione dell'evento syncronize, modifica il proprio periodo adottando un valore comune e riallinea la propria fase temporale rispetto a un tempo globale.

Test plans

  • Si verifichi che ogni lucciola alterni stato ON/OFF con un periodo inizialmente diverso tra le lucciole
  • Si verifichi che dopo 10 secondi venga emesso l'evento syncronize
  • Si verifichi che dopo la sincronizzazione le lucciole lampeggino in modo allineato

Project

Per la visualizzazione dello stato delle lucciole si sfrutta l'attore griddisplay (un servizio separato dedicato alla visualizzazione della griglia) che riceve messaggi di aggiornamento e rappresenta una griglia. 

Dispatch cellstate : cellstate(X,Y,COLOR)
Ogni lucciola invia un messaggio quando cambia stato: 

forward griddisplay -m cellstate: cellstate(X,Y,1)
forward griddisplay -m cellstate: cellstate(X,Y,0)
Per gestire avvio e sincronizzazione si introduce un attore "creator" che crea le lucciole e invia l'evento di sincronizzazione dopo 10 secondi 

QActor creator {
    State s0 initial {
        createActorDynamically(...)
        delay 10000
        emit syncronize
    }
}
