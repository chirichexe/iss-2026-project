Introduction

Realizzazione in Java del VIRTUAL ROBOT

Requirements

Costruire un sistema software che comanda un Differential Drive Robot (DDR) in modo che, 
partendo dalla posizione iniziale HOME, il robot si sposti lungo il perimetro di una stanza rettangolare vuota. 
Il DDR si deve fermare quando tornato in HOME.

Il DDR è anche sensibile ai dati rilevati da un Radar.
Quando il Radar rileva un 'intruso' a una distanza minore di una distanza prefissata ``DMIN``:
  
    
    
  •  se il DDR si sta muovendo , il DDR si ferma fino a che il Radar non rileva più un 'intruso' così vicino 
    • 
    
  •  se il DDR è fermo in HOME, il DDR ruota su sè stesso fino a che un 'intruso' così vicino scompare 
    •

Requirement analysis


Il robot deve essere rappresentato da un attore che esegue il movimento lungo il perimetro della stanza.

Il Radar deve essere rappresentato da un attore che osserva l'ambiente ed è in grado di segnalare la presenza di un intruso vicino

Si utilizza il linguaggio qak per la rappresentazione del modello del sistema



    System ddrroom

    Event obstacle : obstacle(DIST) // il radar ha rilevato un ostacolo a distanza DIST< DMIN
    Event free : free() // non ci sono più ostacoli entro DMIN

    Context ctxroom ip [host="localhost" port=8050]

    QActor ddr context ctxroom {
        State home initial {
            // il robot si trova nella posizione HOME
            // in attesa dell'avvio
        }
        
        State moving {
            // il robot percorre il perimetro della stanza
        }
        
        State stopped {
            // il robot è fermo per presenza intruso
        }
        
        State rotating {
            // il robot ruota su sé stesso
        }
    }

    QActor radar context ctxroom {

        State sensing initial {
            // monitoraggio ambiente
        }
    }


Il comportamento del sistema è caratterizzato da:


    
    
  •  una risorsa esterna che realizza i movimenti reali
    • 
    
  •  acquisizione asincrona dei dati sonar
